# 버전 관리 (Versioning)

## SEMVER (유의적 버전)
저희는 MAJOR.MINOR.PATCH(주.부.수) 버전 관리 체계를 사용하는 SEMVER(유의적 버전 관리) 표준을 따릅니다. 이는 다음과 같은 방식으로 버전을 증가시킨다는 의미입니다.
* **MAJOR(주) 버전:** 호환되지 않는 API 변경 사항이 발생할 때
* **MINOR(부) 버전:** 이전 버전과 호환되는 방식으로 기능을 추가할 때
* **PATCH(수) 버전:** 이전 버전과 호환되는 버그 수정을 할 때

사용자가 알아야 할 **두 가지 개별 버전 번호**가 있습니다. 하나는 API 서비스에 대한 것이고, 다른 하나는 API에서 반환되는 데이터에 대한 것입니다. 서비스에 대한 변경 사항이 적용되어도 PUBG API 데이터에는 영향을 미치지 않을 수 있으므로 버전 관리가 이러한 방식으로 분리되어 있습니다.
* API 서비스의 버전은 `/status` 엔드포인트에서 반환된 데이터에서 확인할 수 있습니다.
* 데이터 버전은 변경 로그(changelog)에서 찾을 수 있습니다.

## 지원 중단 정책 (Deprecation Policy)
지원 중단(Deprecated)되는 필드 및 데이터는 최소 1주일 전에 변경 로그 상단의 **"Upcoming Changes(예정된 변경 사항)"** 섹션에 공지됩니다. 
지원이 중단되면, 해당 필드/데이터는 일반적으로 데이터 보존 기간이 만료될 때까지 14일 동안 하드코딩된 값 `0`으로 유지되며, 그 이후에 완전히 제거됩니다.
