# 속도 제한 (Rate Limits)

PUBG API의 가용성과 안정성을 보장하기 위해 각 API 키에는 분당 보낼 수 있는 요청 수에 제한이 있습니다. 테스트 및 개발 목적의 기본 속도 제한은 **분당 10회 요청(10 RPM)**입니다. 사용 가능한 요청 수를 초과하면 HTTP 429 오류 코드(Too Many Requests, 너무 많은 요청)를 받게 되지만, 1분 이내에 다시 요청을 보낼 수 있게 됩니다.

속도 제한 헤더는 다음과 같이 정의됩니다.
* `X-RateLimit-Limit` - 하루/분당 요청 제한 수
* `X-RateLimit-Remaining` - 해당 시간 창(time window) 내에 남은 요청 수
* `X-RateLimit-Reset` - 속도 제한이 초기화되는 시간 (UNIX 타임스탬프 형식)

## 예상되는 속도 제한 사용량 (Expected Rate Limit Usage)
`/matches` 및 원격 측정(telemetry) 엔드포인트는 속도 제한이 적용되지 않으므로, 일반적인 애플리케이션이 API에 보내야 하는 '속도 제한 대상 요청' 수는 이를 사용하는 사용자/플레이어 수에 정비례해야 합니다. 

예를 들어, 플레이어를 검색할 때마다 애플리케이션은 두 번의 요청을 보내야 합니다 (정보가 아직 캐시되지 않은 경우).
1. `“/players?filter[playerNames]={playerName}”`에 대한 요청 (또는 계정 ID를 이미 아는 경우 `“/players/{accountId}”`)
2. 해당 사용 사례에 시즌 스탯이 필요한 경우 `“/players/{accountId}/seasons/{seasonId}”`에 대한 요청

그 이후 `/matches` 또는 원격 측정 엔드포인트에 보내는 추가적인 요청은 애플리케이션의 API 키 속도 제한에 산정되지 않습니다.

## 더 높은 속도 제한 요청하기 (Requesting a Higher Rate Limit)
기본(10 RPM) 속도 제한이 애플리케이션에 충분하지 않다고 판단되면, "MY APPS" 메뉴에서 **“I NEED A HIGHER LIMIT(더 높은 한도가 필요함)”**을 선택하여 더 높은 한도를 요청할 수 있습니다. 속도 제한 증가 요청을 승인받으려면 **작동하는 샘플과 더 높은 한도가 필요한 이유에 대한 설명이 반드시 필요**하다는 점을 명심해 주세요.

양식을 가능한 한 상세히 빠짐없이 작성해 주시면, 적절한 한도를 제공해 드리기 위해 함께 협력하겠습니다.

애플리케이션에 적절한 속도 제한을 결정하기 위해 당사는 다음과 같은 정보를 필요로 합니다.
* 애플리케이션이 생성하는 API 요청 수
* 요청이 발생하는 시점 (패턴)
* 활성 사용자가 있는 경우, 사용자 수를 보여주는 분석 자료(Analytics)
* 애플리케이션에 필요한 API 요청 수를 줄이기 위해 사용 중인 **캐싱 및 최적화 전략** (추가 정보는 '캐싱 개요' 및 '캐싱 구현' 섹션을 참조하세요)

작업 중인 프로젝트에 진입 장벽(예: 로그인, 권한 등)이 있는 경우, 사이트에 접속하는 방법에 대한 모든 필수 정보를 제공해 주세요.

속도 제한 요청이 승인되면 애플리케이션의 속도 제한이 증가했다는 알림을 받게 됩니다.
